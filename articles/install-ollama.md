## やりたいこと

ローカルでLLMを動かすためのツール「Ollama」のインストール方法を解説します。

## 必要スペック

LLMをローカルで動かすには一定以上のスペックが必要です。
少なくともローカルLLMのモデルサイズ以上のメモリが必要になると思います。
今回、私の手元ではApple M2 24GB, 16GBの2種類のPCで試しました。
qwen3.6はほぼ動かず、gemmma4なら動かすことができました。

## 手順

### 1. Ollamaのインストール

Ollamaは、ローカルでLLMを動かすためのツールです。
公式サイトからインストーラーをダウンロードして、インストールを行います。
https://ollama.com/download

![](https://static.zenn.studio/user-upload/6ae07aebc469-20260508.png)

Macではディスクイメージからアプリをインストールします。

![](https://static.zenn.studio/user-upload/26ac0f13dc51-20260508.png)

### 2. モデルのダウンロード

アプリケーションを起動すると、チャット画面が表示されます。
そこで、モデルを選択します。

![](https://static.zenn.studio/user-upload/ee0e5cce2082-20260508.png)

2026年5月時点での最新バージョンでは、デフォルトで表示されるモデルは、「qwen3.6」と「gemma4」でした。qwenのモデル容量は20GB以上、gemma4は約9GBでした。

今回はgemma4を選択して、最初のプロンプトを送るとモデルのダウンロードが始まります。

![](https://static.zenn.studio/user-upload/be4011b74428-20260508.png)

ダウンロードが完了すると、プロンプトに対する応答が返ってくるようになります。

補足として、雲マークの付いているクラウドモデルはOllamaの有料契約が必要なため、プロンプトを送ってもエラーが返ってきます。

### 3. コマンドの使い方

Ollamaは、コマンドラインからも操作することができます。

#### ① 使えるモデルの一覧表示

現在利用可能なモデルや、ローカルにダウンロード済みのモデルを確認できます。

```bash
$ ollama list
NAME             ID              SIZE      MODIFIED     
gemma4:latest    c6eb396dbd59    9.6 GB    25 hours ago
```

#### ② プロンプトの送信

```bash
$ ollama run gemma4
>>> こんにちは、あなたは誰ですか？
こんにちは！私は**Gemma 4**です。

Google DeepMindによって開発された、オープンウェイトの大規模言語モデルです。

私は、質問に答えたり、文章を作成したり、情報を提供したりすることができます。何かお手伝いできることはありますか？お気軽にお尋ねください。
>>> 日本の歴史について教えてください。
（さらに質問を続ける）
```

#### ③ モデルの削除

ローカルに保存されているモデルが不要になった場合、容量節約のために削除します。

```bash
$ ollama rm gemma4
```

## まとめ

Ollamaは、ローカルでLLMを動かすための便利なツールですが、ハイスペックな環境で動かせない場合はあくまで軽作業向けな印象です。メモリ16GBのパソコンではAIが思考中は動作がやや重くなり並行した作業が若干しづらくなりました。
とはいえ、ローカルでLLMを動かすことができるようになると、クラウドのAPIを介さずに直接モデルと対話できるため、プライバシーに懸念がある内容や機密性の高い内容を扱う場合は便利かと思います。
